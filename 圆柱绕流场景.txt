# Fluid Scientist 二维可配置障碍物流动实验族完整实施方案

## OpenFOAM Foundation 13 仿真、工作站 Python 后处理与对话结果回传

# 1. 本轮任务定位

当前暂不继续扩大到任意流体科研问题、任意物理场和任意动态能力。

本轮先完整实现一个具有代表性、配置范围足够广、能够真实运行和返回结果的实验族：

> 二维长方形流场中，可选底部凸起和一个圆柱；入口、出口、顶部、底部和圆柱边界均可在允许范围内配置；支持恒速流、变速流、空间非均匀入口、压力差驱动、周期压力梯度驱动和顶壁驱动；系统完成自然语言解析、参数确认、OpenFOAM Foundation 13 编译、工作站验证、正式计算、Python 后处理和图片回传。

场景名称统一为：

```text
ConfigurableObstacleFlow2D
```

专用实验规格统一为：

```text
ObstacleFlowExperimentSpecV1
```

本轮最终闭环：

```text
用户自然语言
→ 场景识别
→ 参数和边界提取
→ 物理拓扑判断
→ 必要澄清
→ 用户确认实验规格
→ 几何和网格生成
→ OpenFOAM 13确定性编译
→ 工作站Smoke Test
→ 正式计算
→ 工作站Python后处理
→ 生成流场图和指标
→ 图片和结果回传原对话框
```

本轮成功标准不是“生成了 OpenFOAM 文件”，而是：

```text
自然语言输入
→ 真实工作站计算完成
→ 生成速度/压力/涡量等流场图
→ 用户在对话中直接看到图和观测结果
```

# 2. 第一版场景范围

## 2.1 几何范围

第一版支持：

* 二维长方形计算域；
* 平底通道；
* 可选底部凸起；
* 可选一个圆柱；
* 圆柱可位于凸起上方、前方、后方或其他合法位置；
* 凸起和圆柱参数均可配置；
* OpenFOAM 中使用单层薄三维网格实现二维计算；
* front/back 固定为 `empty`。

支持的组合：

```text
平底 + 无圆柱
平底 + 一个圆柱
凸起 + 无圆柱
凸起 + 一个圆柱
```

数据结构使用：

```text
cylinders[]
```

但 V1 运行时限制最多一个圆柱，为后续多圆柱保留兼容性。

---

## 2.2 物理范围

第一版支持：

* 单相；
* 不可压缩；
* 牛顿流体；
* 等温；
* 层流；
* 稳态 RANS；
* 瞬态层流；
* 瞬态 URANS；
* 水；
* 用户自定义牛顿流体。

第一版暂不支持：

* 多相流；
* 自由液面；
* 热传递；
* 可压缩流；
* 非牛顿流体；
* 多孔介质；
* 流固耦合；
* 圆柱振动；
* 动态变形网格；
* 三维尾迹；
* 用户自定义 OpenFOAM 代码。

不支持的请求返回：

```text
UNSUPPORTED_IN_OBSTACLE_FLOW_V1
```

不得静默忽略。

# 3. 支持的流动拓扑

系统不能默认左右一定是周期边界。

必须首先识别流动拓扑：

```text
INLET_OUTLET
PERIODIC_FORCED
PRESSURE_DIFFERENCE
OPEN_DOMAIN
WALL_DRIVEN
COMBINED_DRIVING
```

---

## 3.1 INLET_OUTLET：入口—出口模式

典型输入：

> 左侧以2 m/s恒定速度流入，右侧压力出口。

合法组合：

```text
left = velocity / mass-flow inlet
right = pressure / open / advective outlet
top = wall / slip / symmetry / freestream / open
bottom = wall
```

主要驱动来自入口速度或流量。

---

## 3.2 PERIODIC_FORCED：周期强迫模式

典型输入：

> 左右为周期边界，施加从左到右的压力梯度。

合法组合：

```text
left/right = periodic pair
forcing = mean pressure gradient 或 body force
top = wall / moving wall / shear stress / slip
bottom = wall
```

左右周期时，不能再分别设置入口和出口。

---

## 3.3 PRESSURE_DIFFERENCE：压力差模式

典型输入：

> 左侧压力较高，右侧压力较低。

结构化为：

```text
left = pressure boundary
right = pressure boundary
pressure_left > pressure_right
```

系统根据：

```text
Δp / L
```

推导平均压力梯度。

不能同时设置左右周期。

---

## 3.4 OPEN_DOMAIN：开放域模式

典型输入：

> 左侧恒速来流，右侧自由流出，上方为自由流边界。

合法组合：

```text
left = velocity inlet
right = pressure/open/advective outlet
top = freestream/open boundary
bottom = wall
```

系统必须检查：

* 顶部距离圆柱和凸起是否足够；
* 出口距离是否足够；
* 是否可能出现回流；
* 开放边界是否使用已验证映射。

---

## 3.5 WALL_DRIVEN：壁面驱动模式

典型输入：

> 顶部壁面向右运动。

或：

> 顶部施加向右剪切应力。

驱动来自：

```text
moving wall
或
shear stress
```

可以配合周期边界，也可以配合入口—出口模式。

---

## 3.6 COMBINED_DRIVING：组合驱动模式

例如：

```text
周期压力梯度
+
顶部运动壁面
```

或者：

```text
恒速入口
+
顶部剪切应力
```

组合驱动必须显式记录，系统不得因为存在速度入口就自动删除压力梯度，也不得自动叠加用户未要求的驱动力。

# 4. 入口能力

入口需要分别表达时间变化和空间分布。

## 4.1 时间类型

第一版支持：

```text
constant
ramp
sinusoidal
piecewise_linear
tabulated
```

### constant

```text
U(t) = U0
```

### ramp

支持：

```text
start_velocity
end_velocity
start_time
end_time
```

### sinusoidal

支持：

```text
mean_velocity
amplitude
frequency
phase
```

### piecewise_linear

支持一组：

```text
(time, velocity)
```

### tabulated

用户上传结构化时序数据，不允许传任意代码。

---

## 4.2 空间类型

第一版支持：

```text
uniform
parabolic
power_law
linear_shear
tabulated_profile
```

### uniform

整个入口速度一致。

### parabolic

适合层流充分发展通道入口。

### power_law

适合经验边界层入口。

### linear_shear

例如：

```text
U(y) = U0 + S y
```

### tabulated_profile

用户提供高度—速度数据表。

---

## 4.3 时间和空间组合

入口定义为：

```text
temporal profile × spatial profile
```

例如：

```text
constant × uniform
ramp × uniform
sinusoidal × uniform
constant × parabolic
ramp × parabolic
tabulated × tabulated
```

V1 不必实现全部组合，但必须在 Schema 中表示支持范围，Validator 必须拒绝未实现组合。

---

# 5. 出口和开放边界能力

## 5.1 pressure_outlet

用户指定参考压力。

适合常规入口—出口流动。

---

## 5.2 open_outlet

表示允许流体自然离开，但其 OpenFOAM 映射必须来自已验证 Boundary Capability。

---

## 5.3 advective_outlet

用于希望扰动以对流形式离开的场景。

---

## 5.4 non_reflecting_outlet

用于尽量降低扰动反射。

如果当前没有 Foundation 13 已验证实现，则不得仅凭名称生成配置，应返回：

```text
CAPABILITY_NOT_VERIFIED
```

---

## 5.5 “自由流出口”的处理

用户说：

```text
自由流出
```

不能直接映射。

生成候选：

```text
pressure_outlet
open_outlet
advective_outlet
non_reflecting_outlet
```

系统结合：

* 是否可能回流；
* 是否关注尾迹扰动；
* 出口距离；
* 当前 Registry 能力；

决定是否自动推荐或询问用户。

# 6. 顶部边界能力

第一版支持：

```text
no_slip_wall
slip_wall
moving_wall
shear_stress
symmetry
freestream
open_boundary
pressure_boundary
```

## 6.1 no_slip_wall

顶部为固定固壁。

## 6.2 slip_wall

法向速度为零，切向不施加无滑移约束。

## 6.3 moving_wall

参数：

```text
velocity_vector
```

## 6.4 shear_stress

参数：

```text
direction
magnitude
unit = Pa
```

需要通过流体物性转换为 OpenFOAM 13 可执行边界。

## 6.5 symmetry

顶部作为对称面。

## 6.6 freestream

需要：

```text
freestream_velocity
```

用于外部来流边界。

## 6.7 open_boundary

允许顶部发生流入或流出，必须使用回流兼容的已验证边界组合。

## 8.1 几何参数

```text
id
```

## 8.2 边界类型

P0 支持：

```text
no_slip_wall

```

```text

```

---

## 8.3 几何合法性检查

新增：

```text
GeometryFeasibilityValidator
```

检查：

* 圆柱不得与凸起重叠；
* 圆柱不得与顶部相交；
* 圆柱不得跨越左右边界；
* 圆柱与壁面间隙不能小于最小网格要求；
* 圆柱不能覆盖观测点；
* 圆柱阻塞比不能超过设定阈值；
* 圆柱与开放边界距离必须足够。

错误示例：

```json
{
  "code": "CYLINDER_INTERSECTS_BUMP",
  "message": "圆柱与凸起发生几何重叠。",
  "suggested_changes": [
    "提高圆柱中心高度",
    "减小圆柱直径",
    "调整凸起位置"
  ]
}
```

# 10. 压力梯度和体力

## 10.1 压力梯度参数

```text
enabled
direction
magnitude
unit
```

允许单位：

```text
Pa/m
m/s²
```

## 10.2 单位转换

物理压力梯度：

```text
dp/dx [Pa/m]
```

转为等效加速度：

```text
a = -(1/ρ) dp/dx
```

如果用户给的是运动学压力梯度：

```text
m/s²
```

则直接作为等效加速度。

## 10.3 方向约定

向右驱动：

```text
pressure decreases along +x
dp/dx < 0
equivalent acceleration points to +x
```

## 10.4 周期压力驱动

左右周期时，必须使用工作站真实验证过的 Foundation 13 实现，例如：

```text
fvModels source
或
verified equivalent body force
```

不得仅凭模型记忆生成。

## 11.1 合法组合示例

### 恒速入口

```text

```

## 11.2 非法组合

必须拒绝：

```text

right = pressure_outlet
```

```text

同时left又是velocity inlet
```

```text

```

```text
所有边界都是静止wall
且无压力梯度、体力或运动壁面
```

```text
恒速入口
同时左右周期
```

```text
pressure-difference mode
同时左右周期
```

# 14. 澄清策略

一轮最多询问三个阻塞问题。

以示例为例，第一轮询问：

1. 顶部是剪切应力还是运动壁面？对应数值和单位是什么？
2. 压力梯度 `4×10⁻⁴` 的单位是 `Pa/m` 还是 `m/s²`？
3. 圆柱的直径和圆心位置是什么？

第二轮询问：

1. 请选择点平均流速或截面平均流速；
2. 给出点坐标或截面位置；
3. 选择 `Ux`、`Uy` 或速度大小。

不要向用户询问：

* `fvSchemes`；
* `fvSolution`；
* OpenFOAM patch 类型；
* PIMPLE 参数；
* 网格字典语法。

# 17. 几何与网格后端

```text

```

必须先选择一个真实可工作的主后端，不要同时半实现多个。

---

## 17.1 推荐后端：Gmsh

若工作站存在：

```text
python3
gmsh Python module
gmshToFoam
```

则优先：

```text
Python Gmsh
→ 构建矩形域
→ 构建底部凸起

```

优点：

* 凸起和圆柱参数化稳定；
* 支持不同轮廓；
* 局部加密方便；
* 边界命名明确；
* 后续扩展多圆柱较方便。

---

## 17.2 备用后端

如果 Gmsh 不可用：


## 18.3 流态与时间模式

系统根据：

* 入口速度；
* 压力梯度；
* 顶壁速度或应力；
* 圆柱直径；
* 通道高度；
* 流体黏度；

估算 Reynolds 数。

推荐规则：

```text
无圆柱且低Re
→ steady laminar

有圆柱且需要尾迹观测
→ transient

较低Re
→ transient laminar

中高Re
→ transient URANS kOmegaSST

只关心平均量且稳态收敛
→ 可先做steady RANS基线
```

第一版不自动选择 LES。

任何从层流切换到湍流的决定都应在实验规格中可见。

# 36. 最终验收标准

## 自然语言

* 支持恒速入口；
* 支持时变入口；
* 支持空间非均匀入口；
* 支持周期压力驱动；
* 支持压力差驱动；
* 支持移动顶壁和剪切顶壁；
* 支持顶部自由流或开放边界；
* 能识别非法边界组合；
* 缺失单位和参数会主动澄清。

## 几何

* 域尺寸可配置；
* 凸起类型、位置、宽度和高度可配置；
* 圆柱位置和直径可配置；
* 圆柱和凸起组合可验证；
* 观测点和截面位置可验证；
* 网格真实通过 `checkMesh`。

## OpenFOAM

* 唯一使用 Foundation 13；
* 使用 `foamRun + incompressibleFluid`；
* 不保留 legacy solver 双重状态；
* 边界和 forcing 来自 VERIFIED 能力；
* 通过串行 Smoke Test；
* 并行任务通过并行 Smoke Test；
* 未验证不能正式提交。

## 工作站运行

* 能正式提交至少4种不同拓扑；
* 能返回实时状态；
* 能识别失败阶段；
* 计算结果保留在工作站；
* 日志和 Case 版本可追踪。

## Python 后处理

* 能生成速度云图；
* 能生成压力云图；
* 能生成涡量图；
* 能生成流线；
* 能生成点速度图；
* 能生成截面平均速度图；
* 能生成圆柱受力图；
* 时变入口能生成响应图。

## 结果回传

* 图片和指标能安全回传；
* 在原对话框中显示；
* 与正确 Run 和 Spec 绑定；
* 修改参数后旧结果不会被误用；
* 后处理失败不影响已完成仿真结果；
* 工作站缺少绘图依赖时返回明确错误。

# 37. 本轮核心原则

本轮不要再回到两个错误方向：

```text
只实现一个固定周期压力驱动Case
```

或：

```text
声称支持任意二维流动，但没有真实验证
```

正确目标是建立一个：

```text
边界可配置
驱动可配置
入口时空形式可配置
凸起可配置
圆柱可配置
观测可配置
OpenFOAM确定性生成
工作站真实验证
Python自动出图
结果直接回传
```

的二维障碍物流动实验族。

最终产品闭环是：

> 用户通过自然语言描述实验，智能体识别流动拓扑和各边界条件，补齐并确认关键参数，生成可运行的 OpenFOAM Foundation 13 Case，在真实工作站完成验证和计算，再调用工作站 Python 生成流场图和观测指标，并把结果直接返回当前科研对话。
